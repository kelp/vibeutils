MKDIR(1) General Commands Manual MKDIR(1)

NAME

mkdir — make directories

SYNOPSIS

mkdir [-pv] [-m mode] directory ...

DESCRIPTION

The mkdir utility creates the directories named as operands, in the order specified, using mode rwxrwxrwx (0777) as modified by the current umask(2).

The options are as follows:

-m mode
Set the file permission bits of the final created directory to the specified mode. The mode argument can be in any of the formats specified to the chmod(1) command. If a symbolic mode is specified, the operation characters “+” and “-” are interpreted relative to an initial mode of “a=rwx”.
-p, --parents
Create intermediate directories as required. If this option is not specified, the full path prefix of each operand must already exist. On the other hand, with this option specified, no error will be reported if a directory given as an operand already exists. Intermediate directories are created with permission bits of rwxrwxrwx (0777) as modified by the current umask, plus write and search permission for the owner.
-v, --verbose
Be verbose when creating directories, listing them as they are created.
-h, --help
Display help message and exit.
-V, --version
Display version information and exit.

The user must have write permission in the parent directory.

EXIT STATUS

The mkdir utility exits 0 on success, and >0 if an error occurs.

EXAMPLES

Create a directory named foobar:

$ mkdir foobar

Create a directory named foobar if it doesn't already exist:

$ mkdir -p foobar

Create multiple directories:

$ mkdir dir1 dir2 dir3

Create nested directory structure, creating any missing parent directories:

$ mkdir -p /tmp/a/deep/directory/structure

Create a directory with specific permissions (700 = read, write, execute for owner only):

$ mkdir -m 700 private_dir

Create multiple directories verbosely, showing each as it's created:

$ mkdir -pv project/{src,tests,docs}

SEE ALSO

chmod(1), rmdir(1), umask(2)

STANDARDS

The mkdir utility is compliant with the IEEE Std 1003.1-2008 (“POSIX.1”) specification.

The -v option is an extension to that specification.
