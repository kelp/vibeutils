NAME

mv — move (rename) files

SYNOPSIS

mv [-finv] source target

mv [-finv] source ... directory

DESCRIPTION

The mv utility moves (renames) files and directories. In its first form, it renames source to target. In its second form, it moves each named source to the destination directory.

The options are as follows:

-f, --force
Do not prompt before overwriting existing files.
-i, --interactive
Prompt before overwriting existing files.
-n, --no-clobber
Do not overwrite existing files.
-v, --verbose
Explain what is being done.
-h, --help
Display help and exit.
-V, --version
Output version information and exit.

If target does not exist, it is created. If target exists and is a directory, the file is moved into that directory.

The mv utility attempts to use the rename(2) system call for efficiency. If the source and destination are on different file systems, mv falls back to copying the file and then removing the original.

EXIT STATUS

The mv utility exits 0 on success, and >0 if an error occurs.

EXAMPLES

Rename a file:

$ mv oldname.txt newname.txt

Move multiple files to a directory:

$ mv file1.txt file2.txt documents/

Move a directory and all its contents:

$ mv old_project/ new_project/

SEE ALSO

cp(1), ln(1), rm(1)

STANDARDS

The mv utility is compliant with the IEEE Std 1003.1-2008 (“POSIX.1”) specification.
