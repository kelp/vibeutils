RM(1) General Commands Manual RM(1)

NAME

rm — remove files or directories

SYNOPSIS

rm [-fIirRv] file ...

DESCRIPTION

The rm utility removes the file or directory entries specified.

By default, rm does not remove directories. Use the -r or -R option to remove directories and their contents recursively.

The options are as follows:

-f, --force
Ignore nonexistent files and arguments, never prompt.
-i
Prompt before every removal.
-I
Prompt once before removing more than three files, or when removing recursively. Less intrusive than -i while still giving protection against most mistakes.
-r, -R, --recursive
Remove directories and their contents recursively.
-v, --verbose
Explain what is being done.
--help
Display help and exit.
--version
Output version information and exit.

SAFETY FEATURES

The rm utility includes advanced safety features:

  • Protection against removing critical system directories
  • Detection of symlink cycles to prevent infinite loops
  • Atomic operations using file descriptors to prevent race conditions
  • Cross-filesystem boundary detection
  • Same-file detection for hard links

EXIT STATUS

The rm utility exits 0 on success, and >0 if an error occurs.

EXAMPLES

Remove a single file:

$ rm file.txt

Remove multiple files with confirmation:

$ rm -i *.log
rm: remove regular file 'error.log'? y
rm: remove regular file 'access.log'? n

Remove a directory and all its contents:

$ rm -rf old_project/

SEE ALSO

rmdir(1), unlink(2)

STANDARDS

The rm utility is compliant with the IEEE Std 1003.1-2008 (“POSIX.1”) specification.

The flags [-I] and [-v] are extensions to that specification.
